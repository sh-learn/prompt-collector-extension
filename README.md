# Prompt Collector to Feishu

Chrome MV3 插件：在 X/Twitter 或普通网页中提取提示词、来源链接和素材，编辑后保存到飞书知识库里的多维表格。

## 功能

- 点击插件图标，在当前网页右侧打开固定采集面板。
- 绑定一个已有的飞书知识库多维表格，读取 Base 下的数据表和字段。
- 切换数据表后按该表字段重新采集并渲染表单，适配文生图、文生视频等不同表结构。
- 自动提取 X 帖子的标题、作者、时间、提示词、多图、可捕获的视频直链和来源链接；普通网页支持选中文本和提示词标记提取。
- X 评论正文会优先使用 GraphQL 响应里的完整文本，减少 `显示更多` 折叠导致的截断；`Seedance参考提示词` 会按视频提示词填充。
- 上传图片附件；X GraphQL 响应里捕获到的 `video.twimg.com` MP4 和普通网页中有直链的 `<video src>` / `<source src>` 视频也会作为附件上传。
- 保存后可直接打开对应飞书数据表。

## 目录

- `extension/`：Chrome 插件本体，包含页面侧边面板、设置页、OAuth 和飞书同步逻辑。
- `test/`：PKCE/config 和内容提取测试。

## 数据表约定

插件不会创建或修改飞书多维表格结构。请先在知识库中创建目标多维表格，并按用途准备不同数据表。

绑定 Base 后，插件按飞书接口返回的字段顺序渲染当前数据表字段，并根据字段名推断可自动填充的内容，例如：

| 内容 | 常见字段名 |
| --- | --- |
| 标题 | `标题`、`名称`、`title` |
| 提示词 | `提示词`、`Prompt`、`正文`、`内容` |
| 来源链接 | `来源链接`、`链接`、`URL` |
| 作者 | `作者`、`博主`、`发布者` |
| 图片 / 视频 | `图片`、`附件`、`视频附件`、`素材附件` |
| 采集时间 | `采集时间`、`保存时间`、`时间` |

字段名固定为 `风格分类` 时，采集面板会隐藏该字段且不会写入，方便交给飞书 AI 或自动化生成。

## 安装

1. 打开 Chrome `chrome://extensions`
2. 开启“开发者模式”
3. 选择“加载已解压的扩展程序”
4. 选择本仓库的 `extension/` 目录

## 首次设置

1. 在飞书开放平台创建应用，准备 `App ID` 和 `App Secret`
2. 打开插件设置页，填写并保存应用信息
3. 复制设置页显示的 Chrome OAuth Redirect URL
4. 在飞书开放平台应用的重定向 URL 中加入该 Redirect URL
5. 回到插件设置页点击“连接飞书”
6. 粘贴知识库中目标多维表格的完整链接，链接必须包含 `table=...`
7. 点击“绑定多维表格”

`App Secret` 仅保存在本机 `chrome.storage.local`。这种方式适合个人自建飞书应用和本地自用插件；公开分发时建议把 token 换取与刷新迁移到后端 token broker。

## 使用

1. 打开 X 帖子或包含提示词的网页
2. 点击插件图标打开页面内侧边面板
3. 面板默认选择 Base 下第一个数据表并自动采集当前页面
4. 切换数据表或点击刷新按钮时，插件会重新采集当前页面
5. 在 X 帖子页检查主帖和当前已加载评论的候选提示词、图片、视频，勾选需要保存的内容
6. 检查并编辑字段内容
7. 点击底部“保存到飞书”

X/Twitter 页面会展示当前 DOM 中已加载主帖与评论的提示词、图片和已捕获视频候选；主帖候选默认勾选，评论候选默认不勾选。勾选多个候选时，它们会合并保存到一条飞书记录。普通网页优先使用选中文本，其次识别 `Prompt:`、`提示词:`、`完整提示词:` 等标记后的正文。字段名包含 `视频提示词`、`Seedance` 时会优先填入视频提示词；字段名包含 `分镜`、`故事板` 时会优先填入分镜/故事板提示词。

飞书同步会按字段类型转换值：URL 字段使用 `{ text, link }`，日期字段写入时间戳，附件字段先上传素材再写入 `file_token`。图片以 `bitable_image` 上传，文件和直链视频以 `bitable_file` 上传，并带当前 Base 的 `drive_route_token`。

## 限制与约束

- 当前主要在 X/Twitter 帖子页验证，X 页面结构变化可能影响正文、作者、图片和当前帖子定位。
- X 评论不会自动展开，也不会自动滚动加载；插件只处理当前页面 DOM 中已经加载出来的主帖和评论。
- X 长评论的完整正文依赖页面加载期间捕获到的 GraphQL 响应；如果插件重新加载前页面已经打开，或响应没有再次发生，仍可能只能读到当前 DOM 里的折叠文本。
- 其他网页只走通用 fallback：优先使用用户选中文本，其次识别 `Prompt:`、`提示词:`、`完整提示词:` 等标记；没有稳定标记时提示词字段可能为空或需要手动整理。
- X 视频通过页面 MAIN world 里的 GraphQL/XHR 观察器捕获 `TweetDetail`、`TweetResultByRestId`、`UserTweets` 等响应中的 `video_info.variants`，只选择最高码率 `video/mp4`。如果视频数据在插件打开前已经加载、X 改了响应结构、只返回 HLS/m3u8，或请求没有再次发生，视频字段可能为空；可尝试重新加载插件后刷新 X 页面。视频直链未捕获时，插件会把 X 视频封面图作为图片候选保留下来。
- 普通网页视频仅支持页面可直接读取到 `<video src>` 或 `<source src>` 的直链场景；DRM、流媒体清单、懒加载或跨域受限资源不保证可上传。
- 图片采集会过滤较小图片并最多保留 12 张素材；头像、装饰图和页面布局图仍可能需要用户检查。
- 插件依赖字段名推断自动填充值。字段名过于个性化、字段类型与内容不匹配，或飞书字段格式要求更复杂时，可能需要改字段名或手动编辑。
- 插件只绑定和写入已有飞书多维表格，不负责创建 Base、数据表、字段、飞书 AI 字段或自动化流程。
- 素材上传需要目标 Base 可访问、附件字段可写、飞书权限完整；单张素材失败时文本仍可能保存，面板状态会提示上传或附件校验结果。
- 面板通过脚本注入到当前网页；Chrome 内置页、扩展页、飞书页面和未授权注入的页面不适合作为采集来源。
- 本地模式会把用户填写的飞书 `App Secret` 和 OAuth token 保存到当前 Chrome profile 的本地存储中，不适合直接作为公开分发方案。

## 飞书权限

飞书应用需要允许用户授权以下 scope：

- `offline_access`
- `wiki:wiki`
- `wiki:wiki:readonly`
- `wiki:node:read`
- `bitable:app`
- `bitable:app:readonly`
- `base:table:read`
- `base:field:read`
- `base:record:create`
- `base:record:retrieve`
- `drive:drive`

如果飞书返回 `permission_violations`，以错误里列出的用户身份权限为准，在开放平台补开后重新连接。
